## Design Philosophy

A portfolio is a product. It has users (recruiters), a goal (get them to reach out), and a 10-second attention window. Engineered accordingly.

Design decisions:
- Dark editorial minimal — eliminates visual noise, keeps focus on the work
- Project cards link to both GitHub and live demos — proof, not claims
- Mobile-first — recruiters open links on phones
- EMERALD gets an interactive node visualization — a transformer model deserves more than a bullet point
